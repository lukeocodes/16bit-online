**Step 2: Implement `packages/server/src/world/regions.ts` (GREEN).**

Exported functions:

1. **`generateRegions(seed, config, landmask, continentMap, continentDefs)`** -- returns `Region[]`:
   - Import `PoissonDiskSampling` from `fast-2d-poisson-disk-sampling`
   - Import `alea` from `alea`
   - Generate Poisson disk points for land regions with minDistance=40, maxDistance=60 within the world bounds
   - Filter points: only keep points that fall on land (landmask value >= 2) for land regions
   - Generate separate Poisson disk points for ocean regions with minDistance=80, maxDistance=120 (larger, since ocean regions are less gameplay-relevant)
   - Filter ocean points: only keep points on ocean (landmask value < 2)
   - Combine land + ocean points into unified Region array
   - For each land region, set `continentId` by looking up `continentMap[z * width + x]` and mapping to the continent def's id (1=first continent def, 2=second, 3=third)
   - For ocean regions, set `continentId` to `"ocean"`
   - If a continent has fewer than 20 regions, add supplemental points at random land positions (seeded) until minimum is met. If more than 30, this is fine -- Poisson disk density should be tuned.
   - Place 0-3 POIs per land region using seeded random, with types sampled from ["ruin", "cave", "landmark", "resource"]. POI positions offset from region center by seeded random within 15 chunks.

2. **`buildRegionLookup(regions, width, height)`** -- returns `Uint16Array`:
   - Create Uint16Array of length `width * height`
   - For each chunk (cx, cz), find nearest region center using squared distance comparison (no sqrt needed)
   - Assign `regionMap[cz * width + cx] = nearestRegion.id`
   - After assignment, compute each region's `chunkCount` by counting entries in the map
   - This is O(chunks * regions) = 810K * ~110 = ~89M comparisons. Per research, this completes in ~80-200ms which is acceptable for startup.

3. **`generateRegionNames(regions, seed)`** -- assigns unique names to each region:
   - Use a seeded PRNG: `alea(\`${seed}-names\`)`
   - Land region names: combine a prefix (terrain-flavored) + suffix (geographic feature). Examples: "Thornwood Vale", "Ashridge Highlands", "Crystalpeak Summit"
   - Ocean region names: directional + body-of-water. Examples: "Northern Passage", "Elven Straits", "Stormbreak Sea"
   - Ensure uniqueness by appending a counter suffix if collision occurs (e.g., "Thornwood Vale II")
   - Prefix lists (at least 30 each for variety):
     - Land prefixes: "Thorn", "Ash", "Crystal", "Shadow", "Iron", "Silver", "Storm", "Frost", "Ember", "Golden", "Moss", "Raven", "Hollow", "Bright", "Dusk", "Dawn", "Copper", "Jade", "Obsidian", "Crimson", "Ancient", "Wild", "Lost", "Pale", "Dark", "White", "Grey", "Stone", "Wind", "Moon"

**Step 2: Implement `packages/server/src/world/biomes.ts` (GREEN).**

Exported functions:

1. **`classifyBiome(elevation, moisture, temperature, continentId)`** -- returns `BiomeType`:
   - Apply continental modifiers BEFORE classification:
     - Elf (`"elf"`): moistMod = +0.15, tempMod = +0.1
     - Dwarf (`"dwarf"`): elevMod = +0.15, tempMod = -0.15
     - Human (`"human"`): no modifiers (most diverse)
     - Ocean (`"ocean"`): return DEEP_OCEAN or SHALLOW_OCEAN based on elevation
   - Clamp modified values to [0, 1]
   - Classification rules (ordered, first match wins):
     ```
     if (e > 0.85) return SNOW_PEAK
     if (e > 0.7) return MOUNTAIN
     if (e > 0.6) return m > 0.5 ? HIGHLAND : TUNDRA
     if (t < 0.2) return TUNDRA
     if (t < 0.35) return m > 0.5 ? BOREAL_FOREST : TUNDRA
     if (m < 0.15 && t > 0.6) return DESERT
     if (m < 0.3) return SCRUBLAND
     if (m > 0.75 && e < 0.3) return SWAMP
     if (m > 0.6) return DENSE_FOREST
     if (m > 0.4) return TEMPERATE_FOREST
     if (e < 0.15) return BEACH
     return TEMPERATE_GRASSLAND
     ```

2. **`classifyBiomes(elevation, moisture, temperature, landmask, continentMap, continentDefs, width, height)`** -- returns `Uint8Array` (biomeMap):
   - For each chunk, determine continentId from continentMap index -> continentDefs[index-1].id (0 = ocean)
   - For ocean chunks: DEEP_OCEAN if landmask=0, SHALLOW_OCEAN if landmask=1
   - For land chunks: call classifyBiome with the chunk's elevation, moisture, temperature, and continentId
   - Inject "wild zones": for each continent, pick a sub-region (seeded) and apply inverted modifiers to create an unexpected biome pocket. Use `alea(\`${seed}-wildzone\`)` to select ~5% of each continent's land area as wild zone with flipped biome modifiers.

3. **`getRegionBiome(regionId, regionMap, biomeMap, width, height)`** -- returns `BiomeType`:
   - Count all biome types among chunks assigned to this region
   - Return the most common biome type (plurality vote)

**Step 3: Implement `packages/server/src/world/worldgen.ts` (GREEN).**

Single exported function:

```typescript
export function generateWorld(seed: number, config?: Partial<WorldConfig>): WorldMap
```

Pipeline:
1. Merge config with defaults from constants (WORLD_WIDTH, WORLD_HEIGHT)
2. Call `generateContinents(seed, fullConfig)` -> `{ landmask, continentMap, continentDefs }`
3. Call `generateElevation(seed, fullConfig, landmask)`
4. Call `generateMoisture(seed, fullConfig, landmask)`
5. Call `generateTemperature(seed, fullConfig, landmask)`
6. Call `generateRegions(seed, fullConfig, landmask, continentMap, continentDefs)`
7. Call `buildRegionLookup(regions, width, height)` -> regionMap. Update each region's chunkCount.
8. Call `classifyBiomes(elevation, moisture, temperature, landmask, continentMap, continentDefs, width, height)` -> biomeMap
9. Call `getRegionBiome` for each region to assign its `.biome` field
10. Call `generateRegionNames(regions, seed)`
11. Build `Continent[]` from continentDefs with computed chunkCounts
12. Return complete `WorldMap` object

Log timing: `console.log(\`[World] Generated in ${elapsed}ms: ${regions.length} regions, ${continents.length} continents\`)`
